The ultimate git sheet-cheat(of doom)

Git can be very confusing and hard to grasp. This is a small little sheet cheat with things you might need. If you need further help... Ask someone who looks smart.

Why should I bother learning git?

Many of the most powerfull tools are hard to learn, git is no different. Learning git will let you more easily work with people on large code bases, move changes over the network, organize your work, structure your team and save you from hard drive failures.(It will also make you objectively cooler <3)

Terminology

Repository/Repo

A project

Commit

A bundle of changes

Staged

Mark changes as "I want to keep these".

Branch

A bundle of commits (A bundle of bundles with changes)

Hash

A series of letters 0-F, usually looks like "25bb8e26be86e67c235f47fc37e578a699c69a1e"

Remote

The project, but on a different computer

Upstream

The "main" repository, usually on someone elses computer.

Make a git repo

# Paste what you copied in the last command

$ git init

# Give in the URL to your remote repo

$ git remote set-url origin "https://gitlab.ida.liu.se/liuid123/reponame"

Show staged changes (to be committed)

$ git diff --staged

Show unstaged changes

$ git diff

Push changes

# Mark what you want to push

$ git add .

# Make sure the right files are staged

$ git status

# Create the commit and describe it

$ git commit -m "Remove bug"

# Send them to someone else's computer

$ git push

Checkout a new branch

$ git checkout -b descriptive-branch-name

Checkout a specific commit

# Copy the hash for the commit you want

$ git log

# Paste what you copied in the last command

$ git checkout 25bb8e26be86e67c235f47fc37e578a699c69a1e

Remove all uncommitted work

$ git reset

Merge a branch

# Move to the branch you want to have both changes, usually "master"

$ git checkout branch-to-merge-from

# Merge in the new branch

$ git checkout topic-branch-to-merge

Solve a merge conflict

# See which files have conflicts

$ git status

# Edit the files and make sure they look like you want.

# Git adds ">>>>>>>>>> HEAD" to show where conflicts are

# When done simply add them and commit

$ git add file_a file_b

$ git commit -m "Descriptive message"

Move un-commited changes to another brnch

# Put the changes in the stash

$ git stash

# Check out the branch to move to

$ git checkout better-branch

# Move the changes from the stash

$ git stash apply
